Building a SwiftUI Card Game with StoreKit In-App Purchases

This tutorial guides you through creating a SwiftUI-based card game app for iOS, focusing on implementing Apple’s StoreKit for in-app purchases. You’ll build an app where users can select from four decks (Classic, Personal, Funny, Dare) and view randomized card questions. The Classic Deck is free, while the others require a one-time purchase to unlock, but after they purchase it they can keep it forever. By the end, you’ll have a functional app with a clean UI and an easy in-app purchase system.

Overview

The demo app is a SwiftUI card game with a sleek gradient UI, where:

  • Users select a deck on the home screen, with icons corresponding to their deck type
  • The Classic Deck is free, while Personal, Funny, and Dare decks show a lock icon and require purchase to unlock.
  • Selecting a deck displays randomized card questions from JSON files in a card view.
  • StoreKit handles purchase prompts and unlocks decks seamlessly.

Overview video of final app: https://www.youtube.com/watch?v=19Zs8Gdee78

Getting Started

Set up your environment:

  • Xcode 16: Download from the Mac App Store or Apple Developer.
  • Swift 5.9: Included with Xcode 16.
  • iOS 18: Target deployment.
  • Simulator/Device: Use Xcode’s Simulator or an iOS device.
  • JSON Files: Add JSON files (classic_deck.json, personal_deck.json, etc.) to your project’s main bundle (with this basic format):

{

    "deck": "Classic Deck",

    "cards": [

        {

            "id": 1,

            "question": "What is your favorite season and why?"

        },

        {

            "id": 2,

            "question": "Do you prefer the beach or the mountains?"

       }

    ]

}
